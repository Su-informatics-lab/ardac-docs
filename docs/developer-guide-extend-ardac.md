# Developer Guide - Extend ARDaC

This guide helps developers customize, integrate, and extend ARDaC services while preserving security, maintainability, and interoperability.

## Extension areas

- Portal components and user-facing pages
- Metadata schemas and validation rules
- Data ingestion and transformation workflows
- API clients and service integrations
- Analysis tools, notebooks, and workflow templates
- Monitoring, reporting, and quality-control utilities

## Development principles

Use version control, automated tests, clear documentation, and review processes for all platform changes. Prefer configuration over custom code when possible.

## Local development

Developers should use approved development environments and avoid testing with sensitive production data unless explicitly authorized.

## Contribution workflow

1. Identify the feature, bug, or integration need.
2. Open an issue or design note.
3. Implement changes in a branch.
4. Add tests and documentation.
5. Submit for review.
6. Deploy through the approved release process.
