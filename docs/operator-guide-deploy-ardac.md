# Operator Guide - Deploy ARDaC

This guide is for teams responsible for deploying, configuring, monitoring, and maintaining ARDaC infrastructure.

## Operator responsibilities

- Provision and configure platform services
- Manage identity, authentication, and authorization settings
- Configure metadata schemas and data ingestion pipelines
- Monitor system health, storage, jobs, logs, and user-facing services
- Apply upgrades, patches, and configuration changes
- Support backup, recovery, and incident response procedures

## Deployment planning

Before deployment, confirm the target environment, security requirements, data governance model, expected user base, storage needs, and integration points.

## Operational checklist

1. Configure identity provider and user roles.
2. Deploy core services and dependencies.
3. Configure portal branding and documentation links.
4. Validate metadata ingestion and search indexing.
5. Test access request and file access workflows.
6. Set up monitoring, logging, alerting, and backups.

## Maintenance

Document all configuration changes, keep runbooks current, and test recovery procedures before production incidents occur.
